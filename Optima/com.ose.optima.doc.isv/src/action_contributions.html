Action Contributions

Many of the views in Optima support action contributions via the org.eclipse.ui.popupMenus extension point and the standard additions menubar path. The System Browser view also supports the custom showViewAdditions menubar path intended for action contributions that will show additional information about the selected System Browser object in another view. Below is a list of the views in Optima that support action contributions and their IDs and object classes:

  • System Browser view

    ID: com.ose.system.ui.views.system.SystemView

    Object classes: com.ose.system.Gate, com.ose.system.Target, com.ose.system.Segment, com.ose.system.Pool, com.ose.system.Heap, com.ose.system.Block, com.ose.system.Process

    Note: All of the above object classes support action filtering on their public boolean is*() methods (e.g. isKilled with the values true or false) and com.ose.system.Target, com.ose.system.Segment, com.ose.system.Pool, com.ose.system.Heap, com.ose.system.Block, and com.ose.system.Process support action filtering on the public boolean has*Support() methods in com.ose.system.Target (e.g. hasStopScopeSupport with the values true or false).

  • Block List view

    ID: com.ose.system.ui.views.block.BlockView

    Object classes: com.ose.system.BlockInfo

  • Process List view

    ID: com.ose.system.ui.views.process.ProcessView

    Object classes: com.ose.system.ProcessInfo

  • Load Modules view

    ID: com.ose.system.ui.views.loadmodule.LoadModuleView

    Object classes: com.ose.system.LoadModuleInfo

  • Dumps view

    ID: com.ose.pmd.ui.view.DumpView

    Object classes: com.ose.system.DumpInfo
