Dump Block Formatters

Identifier:
com.ose.pmd.ui.blockFormatters

Since:
Optima 1.4

Description:
This extension point provides support for user-supplied block formatters for formatting memory and signal blocks in the dump editor.

Configuration Markup:

<!ELEMENT blockFormatter (EMPTY*)>

<!ATTLIST blockFormatter

type        CDATA #REQUIRED

description CDATA #REQUIRED

name        CDATA #IMPLIED

class       CDATA #REQUIRED>

Definition of a block formatter for memory or signal blocks in PMD dump files.


  • type - The type of block formatter; "memory" for a memory block formatter and "signal" for a signal block formatter.
  • description - The description of the block formatter. Used for specifying which memory or signal blocks the block formatter should apply to. For memory blocks, the description is the description field of a memory block. For signal blocks, the description is the decimal signal number of the request signal of a signal block. No value means that the block formatter applies to all blocks of the specified type.
  • name - The name of the block formatter presented by the dump editor user interface.
  • class - The fully qualified name of a public class with a no-arguments constructor that implements the com.ose.pmd.editor.IBlockFormatter interface.

Examples:
The following is an example of a block formatters extension: 
<extension
      point=
"com.ose.pmd.ui.blockFormatters"
>

   
<blockFormatter
         type=
"memory"

         description=
"my-data"

         name=
"My Data"

         class=
"com.example.MyMemoryBlockFormatter"
>

   
</blockFormatter>

   
<blockFormatter
         type=
"signal"

         description=
"80000"

         name=
"My Signals"

         class=
"com.example.MySignalBlockFormatter"
>

   
</blockFormatter>


</extension>
In the example above, there is a memory block formatter for memory blocks with the description my-data and a signal block formatter for signal blocks with the request signal number 80000.

API Information:
See the com.ose.pmd.editor.IBlockFormatter interface in the com.ose.pmd plug-in.

Supplied Implementation:
The com.ose.pmd.editor.HexBlockFormatter and com.ose.pmd.editor.SignalBlockFormatter classes in the com.ose.pmd plug-in are provided as default block formatters for memory and signal blocks in the com.ose.pmd.ui plug-in.
